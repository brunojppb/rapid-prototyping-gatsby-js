Static Site Generators come to the rescue

Ok, I don't want to become a DevOps Engineer. I just want a website.
You basically have those frameworks that can generate static pages for you, without going throught the hustle of caring about infrastructure and so on.

What is Gatsby

  1. Static site generator based on React.
  2. Just React components.
  3. So, hooks are still there.
  4. All the benefits of server-side rendering like SEO.
  5. Hability to plug external datasources like a headless CMS.

Our Little Experiment

At woom we had this interesting project:
  • Migrate our blog frontend to a new platform (Hubspot)
  • But we still want to be flexible enough to be able to hook up different data providers: Headless CMS

Why we decided on Gatsby

Now lets take a look at the main reasons we decided for Gatsby.

Batteries Included

  1. dev and production configuration out-of-the-box
  2. Convention over configuration.
  • No need to figure our how Webpack or Babel works.
  • if I follow the convention, I don't have to figure out config. Good example is routes management

GraphQL

GraphQL integration for static assets like markdown files, but also for remote data sources like a headless CMS.
GraphQL Explorer to view the available data

Rich and flexible plugin ecosystem

Most important plugins provided by Gatsby itself, but also flexible enough to enable us to create custom plugins.
